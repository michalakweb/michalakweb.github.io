My Front End Web Development Setup

An overview of all aspects of my web development workflow, from coding stack to frameworks, task runners, version control, etc.

Local development - MAMP and Local

For my local development I use two applications. Local by Flywheel is fantastic for Wordpress websites, because of its simplicity. Then there is MAMP for everything else.

Text Editor / IDE - Brackets and Visual Studio Code

I started out with Brackets, as it seemed to be easy to setup and use. Once I started learning React it made MUCH more sense to have something like VSC and that's what I'm sticking with currently.

Build tools - Webpack and Gulp

Webpack is now my main module bundler. I use Gulp only in small and simple projects to automatically compile my CSS.

SPA Development - React, Redux, React-Router and Firebase

I'm currently exploring React heavily, along with other tools mentioned here. I find Redux to be the hardest one in terms of implementation.

Version Control - Git and GitHub

Git is an absolutely fantastic tool and it really improves my workflow. I also use GitHub as a place to store my repositories and as a web development portfolio and resume.

CSS Preprocessor - Sass

I like having the ability to define variables, nest and especially organize my project into multiple files, all of which and more I can do with Sass.

CSS Framework - Bootstrap

The first responsive designs I made were without any frameworks, because I wanted to understand the underlying concepts of grids, media queries, viewports, and so on. After that, I realized that the only way to avoid re-writing a lot of the same code, is to use a framework. I chose Bootstrap, because it is well documented.

Content Management System - Wordpress

Currently, I use WordPress for professional projects. WordPress is extremely popular and extendable, I like how powerful it is and how many purposes it can serve. On the downside I know how bloated and slow it gets in time.
